## Trust Boundaries

| Boundary | Description |
|---|---|
| Java request to Python query adapter | Java provides identity and access filter; Python still validates shape and limits. |
| User prompt to retrieval/generation | Guard and router decide whether expensive or unsafe downstream work runs. |
| OpenRouter classifier to route decision | LLM output must be schema-validated and thresholded. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-02-01 | Information Disclosure | prompt injection | mitigate | Input guard hard-rejects known instruction override patterns before retrieval. |
| T-05-02-02 | Denial of Service | broad user prompts | mitigate | Max query length, top-k caps, and unsupported short-circuit. |
| T-05-02-03 | Tampering | LLM classifier output | mitigate | Strict schema validation and confidence threshold before route use. |
</threat_model>
