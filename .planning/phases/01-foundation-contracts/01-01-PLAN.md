<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Contract source to generated code | Malformed schemas can propagate unsafe API/event assumptions into Java and Python generated code. |
| External REST/event clients to contracts | Contracts define validation, auth expectations, and standard error behavior before implementation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `contracts/*.yaml` | mitigate | Keep root contracts as the only source of truth per D-08 and verify generated code from these exact files in Plan 02. |
| T-01-02 | Information Disclosure | API schemas | mitigate | Include auth expectations and ProblemDetail responses in both REST specs so later endpoints have explicit protected/error surfaces. |
| T-01-03 | Repudiation | AsyncAPI metadata | mitigate | Require `eventId`, `correlationId`, `occurredAt`, `sourceService`, and `eventVersion` in every event envelope. |
</threat_model>
