<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host developer machine to containers | Compose exposes local ports and injects environment values into services. |
| App containers to infrastructure containers | Java and Python connect to Postgres and other local services through internal service names. |
| Env examples to real secrets | Examples document required variables without committing secret values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | `.env.example` | mitigate | Use placeholders only and no real credentials; keep `.env` ignored if added. |
| T-06-02 | Spoofing | service credentials | mitigate | Create separate Java and Python database users with scoped ownership in `infra/postgres/init.sql`. |
| T-06-03 | Denial of Service | compose dependencies | mitigate | Add healthchecks and `condition: service_healthy` so app services wait for infrastructure readiness. |
| T-06-04 | Tampering | Makefile wrappers | mitigate | Keep targets as thin wrappers over scripts/config and avoid duplicated contract verification logic. |
</threat_model>
