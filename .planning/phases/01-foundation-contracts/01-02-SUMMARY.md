## Decisions Made

- Used OpenAPI Generator's Spring model generator so Java DTOs are Spring Boot 3/Jakarta compatible, while disabling API and supporting-file generation to keep the module DTO-only.
- Kept generator scripts self-contained with a structured YAML subset parser because Phase 1 has no Python dependency manifest yet.
- Generated Python modules use Pydantic imports when available and include an import-only fallback so the Phase 1 smoke import can run before Plan 01-04 adds ai-service dependency management.
