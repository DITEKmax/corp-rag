<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Contract files to generator scripts | Build scripts transform trusted source YAML into language code consumed by services. |
| Generated code to service compile/import | Java and Python code must fail fast if contract generation is invalid. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Generator scripts | mitigate | Use structured YAML parsing, deterministic outputs, and a single verify command; avoid service-local copied contracts. |
| T-02-02 | Elevation of Privilege | Generated constants | mitigate | Generate routing, queue, exchange, and error-code constants from `contracts/constants.yaml` only, preventing hidden handwritten values. |
| T-02-03 | Denial of Service | Verification command | accept | Local verification may require Maven/Python dependencies; failure is visible before execution continues. |
</threat_model>
