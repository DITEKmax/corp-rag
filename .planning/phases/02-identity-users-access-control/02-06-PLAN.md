## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin role APIs to authorization model | Mistakes can grant or remove broad permissions. |
| JWT permission claims to current role state | Permission changes may lag until token expiry by design. |
| Role update requests to persisted role rows | Concurrent writes need ETag protection. |

| T-02-06-01 | Elevation of Privilege | ADMIN assignment | mitigate | Require both `users.update` and `roles.update` for ADMIN grants. |
| T-02-06-02 | Tampering | role replacement | mitigate | Require If-Match and reject stale ETags for role updates. |
| T-02-06-03 | Denial of Service | last admin | mitigate | Prevent self role modification and last-admin removal. |
</threat_model>
