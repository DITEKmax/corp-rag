<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser cookies to Java API | Cookies authenticate users and must be scoped, protected, and validated. |
| JWT claims to authorization decisions | Claims may become stale and must be bounded by short token lifetime. |
| Refresh token cookie to database state | Reuse or theft must trigger family revocation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Spoofing | access JWT | mitigate | Validate HS256 JWTs through Spring resource-server decoder with a configured 32+ byte secret. |
| T-02-04-02 | Tampering | unsafe cookie requests | mitigate | SameSite Strict plus Origin/Referer validation for unsafe methods. |
| T-02-04-03 | Elevation of Privilege | refresh token reuse | mitigate | Rotate refresh tokens and revoke the token family on reuse. |
| T-02-04-04 | Repudiation | auth events | mitigate | Synchronously write audit events for login/logout/reuse outcomes. |
</threat_model>
