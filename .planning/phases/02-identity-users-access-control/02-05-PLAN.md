<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin user creation to credential delivery | Temporary passwords are sensitive one-time secrets. |
| User profile edits to authorization | Self-service profile access must not become role or admin mutation access. |
| Bootstrap config to persistent admin | First-admin setup can create powerful credentials. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Information Disclosure | temporary passwords | mitigate | Return temporary passwords only once and store only BCrypt hashes. |
| T-02-05-02 | Elevation of Privilege | first-admin bootstrap | mitigate | Require explicit config, run idempotently, and assign only the ADMIN role when no admin exists. |
| T-02-05-03 | Tampering | user profile update | mitigate | Separate self profile update from admin user mutations and forbid role changes in this plan. |
</threat_model>
