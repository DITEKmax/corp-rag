## Dependency Decision

Option A - transformers down-pin:

- Scratch lock with `transformers>=4.57,<5` resolved cleanly.
- Final selected transformer version: `4.57.6`.
- `XLMRobertaTokenizer.prepare_for_model` exists in `transformers 4.57.6`, which matches the FlagEmbedding 1.4.0 reranker path.
- No movement: FlagEmbedding 1.4.0, tokenizers 0.22.2, sentence-transformers 5.5.0, qdrant-client 1.17.1, langgraph 0.2.76, openai 1.109.1, torch 2.12.0, safetensors 0.7.0, accelerate 1.13.0, peft 0.19.1.
- Required transitive movement: `huggingface-hub 1.15.0 -> 0.36.2`, caused by returning to the transformers 4.x line.

Option B - FlagEmbedding upgrade:

- `uv lock --project ai-service --dry-run --upgrade-package flagembedding` found no stable lockfile package change under the existing `flagembedding>=1.4.0,<2.0.0` constraint.
- Prerelease probing changed only resolver mode and did not identify a stable forward-compatible FlagEmbedding replacement appropriate for the project dependency policy.
- Rejected because there was no package version to validate against the current transformers 5.x line.

Final choice:

- Option A was chosen because it is the only empirically available clean resolution.
- It is less ideal than a forward FlagEmbedding upgrade, but it limits movement to transformers plus its Hugging Face hub dependency and keeps unrelated runtime families unchanged.
