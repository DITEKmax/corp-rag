## S4 - Graph Aggregation (05.1-04 historical finding, superseded by 05.1-05)

Probe B "How many vendors are approved in total?" and Probe C "How many vendors does the Compliance department approve?", reranker enabled, vendor doc freshly re-indexed.

Both probes returned:
- `answered=false`, `answer="Accessible evidence is too weak to answer this question reliably."`, `citations=[]`.
- `route=AGGREGATION`, `retrieversUsed=[GRAPH]`, `rerankerUsed=true`, `chunksConsidered=20`, `chunksReturned=5`.
- `confidence` ~4.3e-05 (Probe B) / ~0.0117 (Probe C) - effectively zero.

Anti-lying intent achieved: in Phase 5 the graph route answered with broken citations (`[4]` against a one-element array, `entity:X` snippets). After 05.1 it never emits an out-of-range citation; the final-citation invariant (05.1-03) holds. PH5-UAT-DEF-03 (out-of-range inline refs) is closed in the sense that the model no longer cites nonexistent indexes.

Historical finding PH5.1-DEF-A (root cause confirmed live during 05.1-04): the graph Cypher introduced in 05.1-03 returned `mention.text AS documentText`, but the MENTIONED_IN relationship does not carry a `text` property - confirmed NULL even on a freshly re-indexed vendor document. Neo4j logged `UnknownPropertyKeyWarning: the missing property name is: text`. With no document text on graph candidates, the reranker scored them near zero, confidence collapsed, and the weak-evidence refusal fired. 05.1-05 closed this by resolving graph text from parent chunks and removing relationship-text reads.

Historical 05.1-04 net effect: graph aggregation no longer lied, but no longer answered. 05.1-05 supersedes this state: Probe B/C now answer with valid document-text citations, and graph no-evidence still refuses.
