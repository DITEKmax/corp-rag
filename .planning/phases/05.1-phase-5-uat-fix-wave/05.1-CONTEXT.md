### Observed Code Facts

- `ai-service/uv.lock` currently resolves `flagembedding==1.4.0`, `transformers==5.8.1`, and `qdrant-client==1.17.1`.
- `LocalReranker` already catches exceptions and returns `RERANKER_UNAVAILABLE_WARNING`, but it does not bound model load or scoring time with a reranker-step timeout.
- `OutputGuard` already has unit coverage for invalid refs like `[4]` against too few citations, so F-03 should be reproduced and fixed at the pipeline/final-response assembly level, not by assuming the guard has no invalid-ref test.
- `agent/graph.py` validates against `packed.citations`, then `_select_citations(...)` returns only referenced citation objects without rewriting the answer refs. That can make a previously valid `[4]` invalid in the final response if the final `citations` array contains only one selected citation.
- `graph.py` currently uses `graphPath` as candidate `content`/`snippet`; this explains the F-06 symptom and must be corrected in 05.1-03 while graph citations are made document-backed.
- No `phase5-incident-report.md` fixture was found in the repository; F-04 fixture work is deferred.

<decisions>
## Implementation Decisions

### Fix-Wave Scope

- F-01, F-02, F-03, and the graph-snippet portion of F-06 are treated as a blocking Phase 5.1 insertion before Phase 6.
- Priority order is strict: F-01 -> F-02 -> F-03/F-06 -> live re-UAT.
- F-04, F-05, and F-07 through F-12 are not part of Phase 5.1 unless a fix is required as a direct prerequisite for F-01 through F-03/F-06.

### Reranker Compatibility

- Keep the local `bge-reranker-v2-m3` architecture from verified `D-148` and `D-190`.
- Prefer the smallest compatible dependency change that makes `FlagReranker.compute_score(...)` work.
- Do not silently bump unrelated packages.
- After any FlagEmbedding/transformers change, prove bge-m3 embedding still returns dense 1024 vectors and non-empty sparse weights.

### Reranker Degradation

- Keep `reranker_disabled` distinct from runtime failure.
- Existing `reranker_unavailable` is an acceptable distinct runtime-failure warning; do not rename it unless the executor updates all tests/docs consistently.
- Add a bounded reranker-step timeout shorter than `AI_QUERY_TIMEOUT_SECONDS`.
- Query answers with usable evidence must not return `route=UNSUPPORTED` or `query_timeout` solely because the reranker failed, hung, or was still loading.

### Citation Integrity

- The final response contract is stricter than internal packed context: every `[N]` in `answer` must be valid against the final returned `citations` array.
- The executor may either return the full citation index space used by the prompt, or compact selected citations and rewrite inline refs. Whichever approach is chosen must be tested at the final `QueryResult`/REST response level.
- Graph aggregation answers still require citations; there is no citationless count/list exception.
- Graph citation `quote`/`snippet` fields are user-facing Phase 6 source-viewer data and must resolve to document text, not `entity:X` graph markers.

### Live Re-UAT

- Phase 5.1 is not complete after unit/integration tests alone.
- 05.1-04 must rebuild `python-ai`, re-enable the reranker, rerun the relevant live Phase 5 UAT scenarios, and write follow-up UAT evidence.
- The live run should keep the paid `deepseek/deepseek-v4-flash` model id from UAT for multi-document indexing and demo reliability; the `:free` tier 429 limitation remains F-09 and is not otherwise fixed here.

### Do-Not-Break Gate

After each implementation plan and before 05.1-04 closes Phase 5.1, verify these behaviors remain green:

- bge-m3 dense+sparse embedding, including strict dense 1024 and non-empty sparse token-weight output.
- Atomic Qdrant rollback on indexing failure from `D-94`/`D-95`.
- Input guard prompt-injection and out-of-scope short-circuiting before retrieval.
- Access-filter enforcement and honest no-evidence refusal without filter broadening from `D-180`.
- The mocked Python test suite that was `202 passed / 11 skipped` during Phase 5 UAT stays green, accounting for new tests added by Phase 5.1.

</decisions>
