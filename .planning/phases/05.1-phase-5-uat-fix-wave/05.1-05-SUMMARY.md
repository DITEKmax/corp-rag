# 05.1-05 Summary - Graph Aggregation Text Resolution

## Outcome

PH5.1-DEF-A is closed. Graph aggregation now resolves citeable text from the existing parent chunk store instead of relying on missing Neo4j relationship `text` properties. Probe B and Probe C answer with document-text citations and valid final inline refs.

DEF-B remains open as an info-level Docker reproducibility item. The build still uses Docker Hub `astral/uv:python3.12-bookworm` because ghcr.io is unreachable from the Docker daemon; Python package versions remain pinned through `uv.lock`.

## Implementation

- Removed missing relationship-text reads from graph Cypher (`mention.text` / `evidence.text`).
- Enriched graph candidates in `ParentResolver` from parent chunk content before reranking and context packing.
- Deduped duplicate graph mentions by child chunk before reranking, so one parent chunk is not scored/cited repeatedly.
- Added query-term matching to aggregation graph retrieval. The graph score is now `queryMatchScore`, not hardcoded `0.75`; absent graph evidence returns zero rows.
- Scoped graph-score confidence fallback to document-backed AGGREGATION candidates only.
