## Decisions Made

- **Misconfiguration behavior:** Fail fast. Clamping would hide an operator error and could make UAT evidence ambiguous. Settings validation raises when `max(AI_RERANKER_TIMEOUT_SECONDS, AI_RERANKER_LOAD_TIMEOUT_SECONDS) >= AI_QUERY_TIMEOUT_SECONDS`.
- **Default warm scoring timeout:** 25 seconds. This preserves the approved warm CPU cross-encoder budget while leaving room under the 30 second REST query timeout.
- **Default lazy load timeout:** 28 seconds. This gives cold load slightly more headroom than warm scoring, but still prevents reranking from consuming the full request budget.
- **Cold-load handling:** Cold `BAAI/bge-reranker-v2-m3` load may still exceed local CPU budgets. Plan 05.1-04 must pre-warm the reranker with one untimed query before timing Scenario 3, so the timed evidence measures warm scoring and can demonstrate `rerankerUsed=true`.
