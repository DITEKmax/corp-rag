# Phase 05.1 Plan 03: Citation Repair Summary

**Final answers now share one citation index space with the REST citations array, and graph citations resolve to document text instead of entity markers.**

## Performance

- **Duration:** 25 min
- **Started:** 2026-05-25T22:38:00+03:00
- **Completed:** 2026-05-25T23:03:10+03:00
- **Tasks:** 3
- **Files modified:** 8 code/test files plus plan metadata and this summary

## Accomplishments

- Preserved the full `packed.citations` array in final `QueryResult.citations` so every LLM inline ref uses the same 1-based index space that appeared in the prompt.
- Added a final post-assembly invariant: if the final answer contains any `[N]` where `N < 1` or `N > len(final_result.citations)`, the output is rejected before `answered=true`.
- Added output-guard coverage proving a ref to `[2]` is blocked when only one citation exists.
- Added reranked/reduced top-N pipeline coverage proving an answer citing `[4]` returns at least four final citations and remains valid after reranker ordering.
- Added explicit packed citation evidence tags so the prompt exposes citeable indexes and quotes, not only source IDs.
- Reworked graph citation handling so `entity:X` graph paths remain metadata, while user-facing content, snippet, quote, and citation text prefer document-backed text.

## Citation Contract Decision

Chosen contract: **return the full packed citation index space**.

This was selected instead of compacting selected citations and rewriting inline refs. It is less disruptive, avoids brittle whole-token rewrite logic, and preserves a direct Phase 6 contract: every inline `[N]` maps to `citations[N-1]` in the REST response.

The user-requested rewrite guardrails are satisfied by avoiding rewrite entirely and enforcing the hard requirement after assembly. Hallucinated or out-of-range refs are blocked against the final citations array. Repeated refs and multi-digit refs remain valid by construction because no text mutation occurs. The actual fix gate is the final programmatic invariant over the final REST citations array.
