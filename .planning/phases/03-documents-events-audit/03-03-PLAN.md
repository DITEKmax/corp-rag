<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser upload to Java | File bytes, filename, metadata, and Content-Type are untrusted. |
| Java to MinIO | Object storage is external to the DB transaction. |
| Java metadata to Python event stream | Incorrect event metadata breaks downstream indexing/access filters. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | upload MIME validation | mitigate | Sniff with Tika and use fixed MIME allowlist/extension mapping. |
| T-03-03-02 | Information Disclosure | upload access level | mitigate | Reject uploads above caller's resolved max access level. |
| T-03-03-03 | Denial of Service | multipart uploads | mitigate | Enforce 50MB Spring multipart limit before sniffing. |
| T-03-03-04 | Repudiation | document upload | mitigate | Audit upload details and carry correlation ID into outbox headers. |
| T-03-03-05 | Tampering | object key generation | mitigate | Generate keys from date/documentId/sniffed extension, never original filename. |
</threat_model>
