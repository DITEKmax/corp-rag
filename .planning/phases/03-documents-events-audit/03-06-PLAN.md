<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RabbitMQ to Java consumers | Messages can be duplicated, malformed, late, or redelivered after failure. |
| Python result events to document metadata | Terminal status changes must not violate delete semantics. |
| Consumer transaction to ACK | ACK before commit can lose events; commit before ACK preserves state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06-01 | Tampering | inbound event payload | mitigate | Validate event type and required metadata before processing. |
| T-03-06-02 | Repudiation | consumer audit | mitigate | Set MDC from incoming correlation ID and write detailed indexed/failed audit events. |
| T-03-06-03 | Reliability | duplicate delivery | mitigate | Insert processed_events first and ACK duplicates without side effects. |
| T-03-06-04 | Information Disclosure | deleted document race | mitigate | Skip status update for soft-deleted documents and never resurrect rows. |
| T-03-06-05 | Reliability | business failure | mitigate | Roll back processed_events and let listener retry/DLQ behavior handle requeue. |
</threat_model>
