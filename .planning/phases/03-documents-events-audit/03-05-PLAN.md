<clarifying_notes>
## AMQP Header Propagation On Publish

Task 2 must propagate correlation explicitly across the AMQP boundary; MDC does not cross this boundary automatically.

When calling `rabbitTemplate.convertAndSend`, use a `MessagePostProcessor` to set header `x-correlation-id` from the `outbox_events.headers` JSON, set headers `x-event-type` and `x-event-version`, and set `MessageDeliveryMode.PERSISTENT`.

The outbox row `headers` JSON already contains `correlation_id` captured from MDC when the outbox event was created through `CorrelationIdFilter`; the publisher only transfers that value into AMQP headers.
</clarifying_notes>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Database outbox to RabbitMQ | Broker failures must not lose document events. |
| Generated constants to topology | Queue/routing mismatches strand messages. |
| Scheduled publisher to database | Unbounded scans or retries can overload the DB. |
